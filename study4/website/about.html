Methodology

How we elicit, score, and visualize confidence interval calibration from frontier large language models.

What we measure

Frontier AI models routinely produce confidence intervals alongside their predictions. A well-calibrated 90% confidence interval should contain the true outcome 90% of the time. If a model’s stated 90% intervals only capture the actual value 60% of the time, the model is overconfident — it expresses more certainty than its accuracy warrants.

We measure that gap across four studies using the calibration framework of Soll & Klayman (2004).

Prediction elicitation

Each model receives a structured prompt requesting a point estimate and confidence intervals. The exact prompt wording is shown below for each study. All prompts are sent as a single user message with no system prompt.

Study 2: Multi-Horizon Backtest

Study 2 uses date-blinded prompts (no “Today is…”) to prevent models from recognizing the historical date. Three confidence levels (50%, 80%, 90%) are requested. 

The current price of {ticker} is ${price}.
Please predict the price in {window} days.
Give me three confidence intervals:
- 50% confidence interval: [low, high]
- 80% confidence interval: [low, high]
- 90% confidence interval: [low, high]
Respond ONLY in JSON:
{
  "point_estimate": float,
  "50_ci": [float, float],
  "80_ci": [float, float],
  "90_ci": [float, float],
  "reasoning": string
}

Variables: {ticker} = stock symbol, {price} = current closing price, {window} = days ahead (1, 2, 3, 6, 7, 18, 20, 21, 22). Five independent runs per ticker × horizon.

Study 3: Cross-Domain

Study 3 extends to six domains. Each domain module generates a context string and question text appropriate for the asset type. 

Today is {date}.

{context}

Question: {question_text}

Provide a point estimate and three confidence intervals.
Express all values in {unit}.

Respond ONLY in valid JSON with no markdown fences:
{
  "point_estimate": <number>,
  "50_ci": [<low>, <high>],
  "80_ci": [<low>, <high>],
  "90_ci": [<low>, <high>],
  "reasoning": "<brief reasoning>"
}

Variables: {context} provides current reference values (e.g., “Current BTC price: $94,250.00 USD”), {question_text} is domain-specific (e.g., “What will the closing price of AAPL be tomorrow?”), {unit} = USD, °F, etc.

Study 4: Daily Forecast

Study 4 collects predictions daily with a two-call protocol: first the prediction, then a separate justification call. Only 90% CIs are requested (not 50%/80%) to reduce token costs. Three horizons are requested in a single call. 

Today is {date}.

{context}

Predict the closing price of {item_label} at three future
horizons. For each horizon, provide a point estimate and a 90%
confidence interval. A 90% confidence interval means you are
providing a low value and a high value such that you believe
there is a 90% chance the true closing price will fall between
them. If we asked you to make this kind of estimate 100 times,
the true value should fall inside your interval on about 90 of
those occasions. Choose your bounds accordingly.

Horizons:
  1-day  target date: {date_1d}
  1-week target date: {date_1w}
  1-month target date: {date_1m}

Respond ONLY with valid JSON and no markdown fences:
{
  "1d": {
    "point_estimate": <number>,
    "90_ci": [<low>, <high>]
  },
  "1w": {
    "point_estimate": <number>,
    "90_ci": [<low>, <high>]
  },
  "1m": {
    "point_estimate": <number>,
    "90_ci": [<low>, <high>]
  }
}

After the prediction is received, a second API call requests a free-text justification: 

You previously made the following probabilistic forecast:

{prediction_json}

Please explain your reasoning in 2–4 paragraphs. Address:
1. What drove your point estimates at each horizon?
2. What sources of uncertainty influenced your confidence
   interval widths?
3. Are there specific risks or events that could push outcomes
   outside your stated intervals?

Write in plain English. No JSON, no bullet points.

Calibration metrics

μ (mu) — Meta-knowledge ratio

The primary calibration metric. Based on Soll & Klayman (2004), “Overconfidence in Interval Estimates,” Journal of Experimental Psychology: Learning, Memory, and Cognition.

μ = MEAD / MAD

MAD (Mean Absolute Deviation) is the average absolute difference between the model’s point estimate and the actual outcome, normalized by price:

MAD = mean( |actual − point_estimate| / current_price )

MEAD (Mean Expected Absolute Deviation) is the spread the model expected, derived from its confidence interval width. For a 90% CI assuming normality:

σimplied = (CIhigh − CIlow) / (2 × 1.645)
MEAD = mean( σimplied × √(2/π) / current_price )

Interpretation:

  • μ = 1.0 — perfectly calibrated
  • μ < 1 — overconfident (intervals too narrow)
  • μ > 1 — underconfident (intervals too wide)

Accuracy (normalized MAD)

Average prediction error as a percentage of price. Measures raw forecasting skill, independent of calibration. Lower is better.

Accuracy = mean( |actual − point_estimate| / current_price ) × 100%

90% CI Hit Rate

Proportion of predictions where the actual outcome fell within the stated 90% confidence interval. Target: 90%.

Hit Rate = count(CIlow ≤ actual ≤ CIhigh) / N

Error bars

Charts show 95% confidence intervals (±1.96 × SE):

  • Accuracy SE: std(normalized deviations) / √n
  • μ SE: delta method, propagating uncertainty in both MAD and MEAD
  • Hit rate SE: binomial √(p(1−p)/n)

Limitations

  • Backtest vs. live — Study 2 uses date-blinded prompts for historical dates. Models may behave differently in a live setting (Studies 1, 3, 4).
  • Normality assumption — the μ calculation assumes normally distributed errors when converting CI width to implied σ. Fat-tailed domains (crypto, commodities) may violate this.
  • Small n in some domains — Study 3 commodities (n=5/model) and forex (n=8/model) produce wide confidence intervals on the metrics themselves.
  • Model versioning — Studies 1–3 used model snapshots from early 2026. Study 4 uses updated versions. Direct comparison across studies should account for this.

References
