About this study

Study 4 of an ongoing empirical research project on overconfidence in frontier LLMs.

What we're measuring

Frontier AI models routinely produce confidence intervals alongside their predictions. A well-calibrated 90% confidence interval should capture the true outcome 90% of the time. If a model's stated 90% CIs only contain the true value 60% of the time, the model is overconfident — it expresses more certainty than its accuracy warrants.

This study tracks that gap continuously and publicly, across four domains and three prediction horizons.

The index metric

90% CI Hit Rate = proportion of predictions where the true outcome fell within the model's stated 90% confidence interval, over a rolling 30-day window.

Target: 0.90 (90%). Below 0.90 = overconfident; above 0.90 = underconfident.

We also compute ECE (Expected Calibration Error) = mean |stated coverage − empirical coverage| across 50%, 80%, and 90% confidence levels, and a Brier score = ((point_estimate − actual) / reference_value)² for point-estimate accuracy.

Study design

Each weekday at 16:30 UTC (after US market close), we query three AI models with the same set of 40 prediction items. Each model produces a point estimate and three confidence intervals (50%, 80%, 90%) for three future horizons. A separate follow-up call collects the model's free-text reasoning — always after the prediction, never combined with it.

When each prediction's horizon elapses, we fetch the resolved outcome from public data sources and score the prediction. The index updates daily.

Models

  • Claude — claude-sonnet-4-6 (Anthropic)
  • GPT-4o — gpt-4o-2024-11-20 (OpenAI)
  • Gemini — gemini-2.5-flash (Google)

Model version strings are pinned and recorded verbatim in every prediction file. Any mid-study model update is logged as a version-change event and noted in the data.

Domains and items

  • Stocks (10) — SPY, AAPL, MSFT, GOOGL, NVDA, AMZN, META, TSLA, JPM, BRK-B. Actuals from Yahoo Finance (closing price).
  • Crypto (10) — BTC, ETH, SOL, BNB, XRP, ADA, AVAX, DOGE, DOT, LINK. Actuals from CoinGecko public API.
  • Forex (10) — EUR/USD, GBP/USD, USD/JPY, AUD/USD, USD/CAD, USD/CHF, NZD/USD, USD/CNY, USD/MXN, USD/SGD. Actuals from Yahoo Finance.
  • Weather (10 cities) — New York, London, Tokyo, Sydney, São Paulo, Lagos, Mumbai, Toronto, Dubai, Berlin. Metric: mean daily temperature in °F. Actuals from Open-Meteo historical archive (free, no API key).

Prediction horizons

  • 1-day — target date is prediction date + 1 calendar day
  • 1-week — target date is prediction date + 7 calendar days
  • 1-month — target date is prediction date + 30 calendar days

For financial items, if the target date falls on a weekend or non-trading day, the next available trading day close is used and the adjustment is logged.

Ramp-up period

1-day outcomes are available from Day 1. 1-week outcomes from Day 7. 1-month outcomes from Day 30. The index for each horizon is withheld until sufficient data is available.

Limitations

  • Fixed item set — the same 40 items are predicted every day. Models may implicitly "learn" the item list over time, though each query is stateless (no conversation history).
  • Single daily snapshot — predictions are made once per day, not continuously. Intraday volatility is not captured.
  • Free API rate limits — CoinGecko's free tier imposes rate limits. On rare occasions a crypto actual may not be retrieved; affected predictions are marked as status=failed and excluded from hit rate calculations.
  • Model version pinning — API providers may silently update models behind fixed version strings. We log exact model IDs per prediction but cannot fully control for undisclosed upstream changes.
  • No replication across API calls — each prediction is made once (one run per item per model per day). Stochasticity in model outputs is not averaged.

Data and code

All prediction files, actuals, scored data, and analysis code are available in the project's GitHub repository. This site is a read-only view on those data files.
