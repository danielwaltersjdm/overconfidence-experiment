About This Study

Study 4 of an ongoing empirical research project on overconfidence in frontier large language models.

What we measure

Frontier AI models routinely produce confidence intervals alongside their predictions. A well-calibrated 90% confidence interval should contain the true outcome 90% of the time. If a model's stated 90% intervals only capture the actual value 60% of the time, the model is overconfident — it expresses more certainty than its accuracy warrants.

This study tracks that gap continuously and publicly, using the calibration framework of Soll & Klayman (2004), across 200 stocks and three time horizons.

Predictions / day

Each weekday after US market close, we query three AI models with the same set of 200 stocks. Each model produces a point estimate and a 90% confidence interval for three future horizons (1-day, 1-week, 1-month).

A separate follow-up API call then collects the model's free-text reasoning for its prediction. The justification is always collected after the numeric prediction, never combined with it, to avoid the reasoning influencing the numbers.

When each prediction's horizon elapses, we fetch the resolved closing price from Yahoo Finance and score the prediction. The index updates daily.

Calibration metrics

μ (mu) — Meta-knowledge ratio

The primary metric on the dashboard. Based on Soll & Klayman (2004), “Overconfidence in Interval Estimates,” Journal of Experimental Psychology: Learning, Memory, and Cognition.

μ = MEAD / MAD, where:

μ = MEAD / MAD

MAD (Mean Absolute Deviation) is the average absolute difference between the model's point estimate and the actual outcome, normalized by price:

MAD = mean( |actual − point_estimate| / current_price )

MEAD (Mean Expected Absolute Deviation) is the spread the model expected, derived from its confidence interval width. For a 90% CI assuming normality:

σimplied = (CIhigh − CIlow) / (2 × 1.645)
MEAD = mean( σimplied × √(2/π) / current_price )

Interpretation:

  • μ = 1.0 — perfectly calibrated. The model's stated uncertainty matches the actual spread of outcomes.
  • μ < 1 — overconfident. Confidence intervals are too narrow relative to actual prediction error.
  • μ > 1 — underconfident. Confidence intervals are wider than necessary.

Accuracy (normalized MAD)

The average prediction error as a percentage of the current price. This measures raw forecasting skill, independent of calibration. Lower is better.

Accuracy = mean( |actual − point_estimate| / current_price ) × 100%

This is mathematically identical to the MAD in the μ formula, just displayed as a standalone percentage. A model can have good accuracy (low error) but poor calibration (intervals that don't match the error), or vice versa.

90% CI Hit Rate

The proportion of predictions where the actual outcome fell within the model's stated 90% confidence interval. Target: 90%. Below 90% = overconfident. Above 90% = underconfident.

Hit Rate = count(CIlow ≤ actual ≤ CIhigh) / N

Additional metrics (in scored data)

Expected Calibration Error (ECE)

ECE = |stated coverage − empirical coverage|

For the 90% CI level. Lower is better. ECE = 0 means perfect calibration.

Brier score (normalized)

Brier = ((point_estimate − actual) / current_price)2

Measures point-estimate accuracy as squared error. Normalizing by current price makes the score comparable across stocks at different price levels. Lower is better.

Limitations

  • Single asset class — this study tracks US equities only. Cross-domain generalization (crypto, forex, weather) was explored in Study 3.
  • Single daily snapshot — predictions are collected once per day, not continuously. Intraday volatility is not captured.
  • Model version pinning — API providers may silently update models behind fixed version strings. We log exact model IDs per prediction but cannot fully control for undisclosed upstream changes.
  • No replication — each prediction is made once per item per model per day. Stochasticity in model outputs is not averaged out.
  • Normality assumption — the μ calculation assumes returns are normally distributed when converting CI width to implied standard deviation. Fat-tailed distributions would affect the MEAD estimate.
