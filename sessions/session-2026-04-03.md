Created `study4/` from scratch with all required deliverables:

#### Configuration
- `config.yaml` — 4 domains × 10 items, 3 models (pinned IDs), 3 horizons, weekend config, model version change log
- `requirements.txt` — anthropic, openai, google-genai, yfinance, requests, pyyaml, pandas, numpy, python-dotenv, rich

#### Python pipeline (all run from `study4/`)
- `collect_predictions.py` — collects prediction + justification (always 2 separate calls) for all 40 items × 3 models. Weekend skip logic, CoinGecko rate limiting, idempotent (skips existing files), `--dry-run` / `--domain` / `--model` flags
- `fetch_actuals.py` — scans all prediction files, fetches resolved actuals per horizon per item. Stocks/forex via yfinance (next-trading-day adjustment), crypto via CoinGecko, weather via Open-Meteo archive. Writes `data/actuals/actuals.csv`
- `score.py` — computes hit_50/80/90, Brier score, ECE per model × domain × horizon. Writes `data/results/scored.csv` and `summary.csv`. Consistent formula with Studies 1–3
- `export_data.py` — generates rolling 30-day index and time series. Writes `website/data/rolling_index.json` and `time_series.json`. Handles ramp-up (insufficient_data status for elapsed horizons)

## Key Design Decisions

- Follows Study 3 conventions exactly: `from google import genai`, `ci_50_low`/`ci_50_high` naming, `current_value`, `call_with_retry`
- Justification is always a **separate API call** after prediction — enforced structurally (second call references the prediction response object)
- `PROMPT_VERSION = "1.0.0"` constant in collect_predictions.py for prompt change tracking
- actuals.csv design: one row per (model, item, domain, pred_date, horizon) — same model per row so scoring join is trivial
- Rolling index handles ramp-up: 1d active from day 1, 1w from day 7, 1m from day 30; `days_until_first` field for website display
- Weather reference value = yesterday's mean from Open-Meteo archive (matches actuals format, avoiding current vs. daily mean mismatch)
- Forex safe IDs: slashes removed (`EUR/USD` → `EURUSD`) for filenames
