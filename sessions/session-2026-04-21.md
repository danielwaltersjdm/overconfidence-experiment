# Session 2026-04-21

Rolling session spanning 2026-04-10 through 2026-04-21: website redesign,
pipeline hardening, and deployment stabilization.

## Key Changes

### Website redesign — AI Calibration Index (`study4/website/`)

- **Landing page** (`index.html`) is now the **Daily Forecast Tracker**
  (Study 4 longitudinal), replacing the previous Study-2-as-landing design
- **Renamed pages** to content-descriptive labels:
  - `index.html` — "Daily Forecast Tracker" (was: unlabeled Study 4)
  - `backtest.html` — "Horizon Backtest" (new file, was part of old `index.html`)
  - `crossdomain.html` — "Cross-Domain" (new page, Study 3)
  - `about.html` — "Methodology" (was "About")
  - `researcher.html` — bio page for Daniel Walters (new)
  - `tracker.html` — redirect stub → `index.html`
- **Hero titles rewritten** to be descriptive and neutral:
  - Daily Forecast: "Daily Calibration Tracker"
  - Horizon Backtest: "Calibration Across the Horizon"
  - Cross-Domain: "Confidence Across Domains"
- **Model cards**: 90% CI Hit Rate is the top-line metric on all pages
  (was μ); μ and accuracy moved to secondary details row
- **Error bars** added to all three charts (Daily Forecast, Backtest, Cross-Domain)
- **Domain filter** dropdown added to Cross-Domain page
- **Aggregate view** on Backtest: new "All Horizons" option in horizon dropdown
  (default) shows n=18,832 combined stats
- **Study 3 integration**: new Cross-Domain page with 5 domains (stocks, crypto,
  forex, weather, commodities — NBA dropped as mostly unresolved)
- **Methodology page** reorganized: calibration metrics moved to top (above
  study designs); exact API prompt wording shown in dark code boxes for each
  study; Study 1 removed from website references (no presence on site)

### Data pipeline fixes

- **Delisted tickers removed** from Study 4 config: FI, HES, MMC
  (all returned "No data found, symbol may be delisted" from Yahoo Finance
  on every collection run)
- **Replacements added**: NXPI (NXP Semiconductors) for FI, FICO (Fair Isaac)
  for MMC. HES dropped without replacement (was 201 → now exactly 200 stocks)
- **Gemini retry logic hardened** in `collect_predictions.py`:
  - 5 retries (up from 3)
  - 30s+ initial backoff with exponential increase for 503/rate-limit errors
    (up to ~8 min total vs. previous ~16s)
  - Previously 270/402 Gemini files failed on Apr 8 due to API overload
- **Failed-file re-collection**: `collect_predictions.py` now re-attempts files
  whose existing JSON has `status != "collected"` instead of skipping them

### GitHub Actions workflow fixes

- **Export to Website workflow**: `git add -f study4/website/data/` (was failing
  since Apr 9 because `website/data/` is in `.gitignore`)
- **Deploy Pages workflow**: added `workflow_run` trigger listening to
  "Export to Website" completion. Previously the GITHUB_TOKEN commits from
  the export workflow were not triggering downstream workflows (GitHub safety
  feature preventing infinite loops), so deploys stopped firing after exports
- Full pipeline now runs cleanly end-to-end every trading day:
  Daily Collect (16:30 UTC) → Daily Score (18:00 UTC) → Export to Website
  (workflow_run) → Deploy Pages (workflow_run)

### Data exports

- **`generate_from_study2.py`** rewritten to output to `website/data/study2/`
  (separate from Study 4 data — was mixing with Study 4's `rolling_index.json`

  Study 2's 9-horizon data)
- **`generate_from_study3.py`** created: exports Study 3 scored data with

- **`export_data.py`** now includes SE values in `time_series.json` so the

## Decisions

- Landing page is the live Study 4 tracker, not a historical study (the
  ongoing data collection is the most interesting view)
- Backtest cards default to "All Horizons" aggregate (n=18,832) — individual
  horizons still selectable from dropdown
- 90% CI Hit Rate is the primary card metric across all pages (more intuitive
  than μ for non-technical readers)
- Cross-Domain hero wording kept neutral (no directional claims about
  over/underconfidence — let the data speak)
- NBA excluded from Cross-Domain (predictions mostly unresolved — ESPN API
  never returned completed status for the game window)
- `website/data/` stays in `.gitignore` but export workflow force-adds via
  `git add -f` (local runs rebuild from scored.csv; CI commits the generated
  JSON so GitHub Pages can serve it)
